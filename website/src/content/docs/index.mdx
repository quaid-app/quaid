## What makes Quaid different

<div class="gb-guide-grid gb-guide-grid--two">
  <GuideFeatureCard variant="line" icon="seti:db" title="Append-only by design">
    <p>Memories accumulate. Nothing is rewritten. A head pointer resolves queries to current truth, but the full history is always accessible. You can ask what you believed before, trace how a view evolved, and audit the chain at any time.</p>
  </GuideFeatureCard>
  <GuideFeatureCard variant="line" icon="open-book" title="Truth is human-readable">
    <p>Markdown defines what's true. SQLite stores it deterministically. Embeddings only index it. The index is rebuildable; the truth is not. Every layer is inspectable, and you can read every page with a text editor.</p>
  </GuideFeatureCard>
  <GuideFeatureCard variant="line" icon="laptop" title="One binary, no cloud">
    <p>Compute and storage on your machine. The embedding model ships in the binary on the airgapped channel. No container, no Python runtime, no service mesh, no API keys. Drop one statically-linked file on a laptop, a Pi, or an air-gapped workstation and it works the same.</p>
  </GuideFeatureCard>

## Skills, in the binary

<Card title="Workflows ship inside the binary, not in a prompt" icon="puzzle">
  Most agent tooling ships workflow logic in the cloud or stuffed into a prompt. Quaid ships eight skills as Markdown files inside the static binary - ingest, query, maintain, enrich, briefing, alerts, research, upgrade. They extract to <code>~/.quaid/skills/</code> on first run, and any <code>SKILL.md</code> dropped in your working directory overrides the default. The agent reads what's on disk; you read what the agent reads.
</Card>

## Get running in seconds

Three commands. No API keys. No cloud.

```bash
# Initialize a memory
quaid init ~/memory.db

# Attach your notes as a live-sync collection
quaid collection add notes ~/notes --db ~/memory.db

# Expose Quaid's MCP tools over stdio (22 in `v0.18.0`; this branch expands to 24 with the unreleased `v0.19.0` extraction + correction follow-on)
QUAID_DB=~/memory.db quaid serve
```
