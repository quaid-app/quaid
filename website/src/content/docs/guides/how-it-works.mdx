## The compiled knowledge model

<div class="gb-guide-split">
  <div class="gb-guide-stack gb-guide-stack--tight">
    <p class="gb-guide-section-intro">
      GigaBrain adapts the compiled knowledge model so every page has a current
      truth layer and an append-only evidence trail. You get quick answers
      without losing the chain of events that produced them.
    </p>
    <div class="gb-guide-grid gb-guide-grid--two">
      <GuideFeatureCard icon={"approve-check-circle"} title={"Compiled truth"}>
        <p>
          The top of the page is the fast-answer layer — always current,
          rewritten when new information arrives, and optimized for retrieval.
        </p>
      </GuideFeatureCard>
      <GuideFeatureCard icon={"list-format"} title={"Append-only timeline"}>
        <p>
          The lower half is your evidence log — never rewritten, always dated,
          and ideal for verifying where a conclusion came from.
        </p>
      </GuideFeatureCard>
    </div>
  </div>
  <GuideVisualPanel
    variant={"diagram"}
    label={"Markdown → SQLite → hybrid retrieval"}
    note={

</div>

## Engine layout

<div class="gb-guide-grid gb-guide-grid--two">
  <GuideFeatureCard icon={"document"} title={"Metadata via SQLite"}>
    <p>
      Pages, links, tags, assertions, and metadata all live in a single SQLite
      database with ACID semantics, portability, and zero setup overhead.
    </p>
  </GuideFeatureCard>
  <GuideFeatureCard
    icon={"rocket"}
    title={"Local embeddings via Candle"}
    variant={"accent"}
  >
    <p>
      GigaBrain uses Rust-native Candle execution to run BGE-small locally with
      CPU-friendly SIMD performance and no cloud dependency.
    </p>
  </GuideFeatureCard>
</div>

<div class="gb-guide-surface gb-guide-stack gb-guide-stack--tight">
  <div>
    <h3 class="gb-guide-subtitle">FTS5 hybrid search</h3>
    <p class="gb-guide-section-intro">
      Semantic retrieval and keyword search are fused instead of forced apart.
      FTS5 handles lexical precision, vectors add semantic recall, and
      Reciprocal Rank Fusion merges the final ranking.
    </p>
  </div>
  <div class="gb-guide-split gb-guide-split--compact">
    <div>
      <p>
        That means <code>gbrain search</code> is ideal for exact terms, while{" "}
        <code>gbrain query</code> can answer concept-level questions without
        leaving the local database.
      </p>
    </div>
    <div class="gb-guide-grid gb-guide-grid--two gb-guide-grid--dense">
      <GuideMetricCard
        value={"FTS5"}
        label={"Keyword index"}
        note={"Native SQLite full-text search with predictable, local latency."}
      />
      <GuideMetricCard
        value={"RRF"}
        label={"Result fusion"}
        note={
          "Keyword and semantic rankings are combined instead of choosing one winner."
        }
      />
    </div>
  </div>
</div>
