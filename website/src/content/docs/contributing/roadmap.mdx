Quaid moves through three arcs. Each arc is a directional bet, not a feature list. The phases inside each arc have hard ship gates: no phase begins until the previous one passes.

<Aside type="tip" title="The shape of the bet">
**Arc 1** ships a private, offline knowledge runtime that holds. **Arc 2** turns that runtime into an agent that compounds knowledge across sessions. **Arc 3** turns that compounding into something that genuinely thinks. Each arc is independently useful; together they are the whole product.
</Aside>

---

## Arc 1 - Local memory that holds <Badge text="Mostly shipped" variant="success" />

The foundation. Storage that doesn't lose data, retrieval that doesn't hallucinate, and a sync engine that follows your editor without dropping writes. Most of this arc is in users' hands today.

## Arc 2 - Agents that compound <Badge text="Next" variant="note" />

The strategic bet. The runtime can store and retrieve, but it can't yet extract structured facts from conversation turns - and that's the gap the benchmarks measure. Arc 2 closes it with a local SLM and the entity layer that follows from it.

<Aside type="note" title="Honest about today">
**LongMemEval:** Quaid 0.0% → Mem0 v3 93.4%
**LoCoMo:** Quaid 0.1% → Mem0 v3 91.6%

The gap is architectural, not a retrieval failure. Quaid stores whole conversation turns. The benchmarks ask specific factual questions. Without fact extraction at write time, retrieval cannot surface structured answers. Phase 5 ships the local SLM that closes it - airgapped, downloaded once, never calls home. **Targets after implementation: LongMemEval > 40%, LoCoMo > 40%.**
</Aside>

### Phase 5 - Conversation memory + SLM extraction

Outcome: an agent that remembers the right things from a conversation turn, not just the turn itself.

<CardGrid>
  <Card title="Namespace isolation" icon="puzzle">
    <Badge text="No dependencies" variant="success" />

    Per-namespace memory isolation for multi-agent and multi-session contexts. Drops LongMemEval benchmark runtime from 3-6 hours to under 30 minutes.

    [→ GitHub #137](https://github.com/quaid-app/quaid/issues/137)
  </Card>
  <Card title="Large corpus performance" icon="rocket">
    <Badge text="No dependencies" variant="success" />

    Optimize for 10K+ document stores. Current import: ~437s for 350 docs. Target: under 60s. Validate Quaid at the scale real users need (GBrain operates at 75K files).

    [→ GitHub #134](https://github.com/quaid-app/quaid/issues/134)
  </Card>
  <Card title="Conversation ingestion + SLM extraction" icon="star">
    <Badge text="Core feature" variant="tip" />

    Ingest conversation turns, extract structured facts using a local SLM, store as typed pages. Non-blocking background queue, fully airgapped.

    **Recommended model:** Phi-3.5 Mini (MIT, ~2GB) as default. Configurable - swap to Gemma 3 1B/4B for quality.

    [→ GitHub #105](https://github.com/quaid-app/quaid/issues/105)
  </Card>
  <Card title="Contradiction resolution" icon="warning">
    <Badge text="Depends on #105" variant="caution" />

    Supersede and version stale facts. "I used to work at X, now Y" resolves correctly rather than keeping both facts active. Version chains, temporal-latest query mode.

    [→ GitHub #135](https://github.com/quaid-app/quaid/issues/135)
  </Card>
</CardGrid>

### Phase 6 - Knowledge graph

Outcome: query by entity, not just text. Once Phase 5 extracts entities, the graph is a thin layer on top.

<CardGrid>
  <Card title="Entity extraction at write time" icon="document">
    <Badge text="Graph foundation" variant="tip" />

    Extract people, companies, and concepts at ingest time using zero-LLM heuristics. Coordinates with #105 - shares the same SLM extraction pipeline.

    [→ GitHub #107](https://github.com/quaid-app/quaid/issues/107)
  </Card>
  <Card title="Self-wiring knowledge graph" icon="random">
    <Badge text="Depends on #107" variant="caution" />

    YAML frontmatter as first-class graph edges. Documents auto-link via wikilinks and tags. From GBrain data: +28% graph search performance, -53% noisy results.

    [→ GitHub #72](https://github.com/quaid-app/quaid/issues/72)
  </Card>
  <Card title="Graph traversal query API" icon="magnifier">
    <Badge text="Depends on #72, #107" variant="caution" />

    Query by entity: `memory_graph_query(entity, hops)`. Find all pages connected to X, find concepts that link two people, 2-hop traversal in under 500ms.

    [→ GitHub #133](https://github.com/quaid-app/quaid/issues/133)
  </Card>
  <Card title="Multi-hop graph traversal" icon="right-arrow">
    <Badge text="Depends on #133" variant="caution" />

    Configurable 1-3 hop traversal, edge type weighting, relationship scoring combined with semantic similarity.

    [→ GitHub #74](https://github.com/quaid-app/quaid/issues/74)
  </Card>
</CardGrid>

## Arc 3 - Memory that thinks <Badge text="Later" variant="note" />

The features that turn a memory store into a memory system. With the graph built and conversation memory working, the runtime can be proactive instead of just responsive.

### Phase 7 - Intelligence layer

Outcome: memory that updates itself when new content arrives, deduplicates noise at scale, and compresses what it sends to an LLM.

<CardGrid>
  <Card title="Active memory enrichment" icon="information">
    <Badge text="Depends on #107, #133" variant="caution" />

    When new content arrives, existing entity memories update proactively. GBrain v0.23.0's headline feature: "Your brain actually gets enriched from your conversations."

    [→ GitHub #136](https://github.com/quaid-app/quaid/issues/136)
  </Card>
  <Card title="Noise reduction + deduplication" icon="list-format">
    Result deduplication and relevance filtering. Implementable at any point but most valuable when the corpus is large.

    [→ GitHub #75](https://github.com/quaid-app/quaid/issues/75)
  </Card>
  <Card title="Context compression (REFRAG)" icon="compress">
    Compress retrieved chunks before sending to an LLM. Reduces tokens and cost while maintaining quality. High value at production scale.

    [→ GitHub #76](https://github.com/quaid-app/quaid/issues/76)
  </Card>
</CardGrid>

## Deferred

These are known design choices, not oversights.

| Feature | Status | Reasoning |
|---------|--------|-----------|
| npm, Homebrew, winget | Planned | Registry ownership and release automation not yet in place |
| WASM compilation | Possible | Strong Rust support, not yet a priority |
| Collaborative / multi-user | Not planned | Single-writer by design. No auth, RBAC, or CRDTs. |
| LLM-assisted contradiction detection | Deferred | Binary stays dumb. Cross-page reasoning lives in the `maintain` skill. |
| Room-level palace filtering | Deferred | Wing-only until benchmarks prove room-level helps. |
