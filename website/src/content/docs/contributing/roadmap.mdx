Quaid is built in phases. Each phase has a hard ship gate — no phase begins until the previous one passes. The roadmap below reflects current priorities based on benchmark results, competitive analysis, and user feedback.

<Aside type="tip" title="The vision">
Phase 1 + 2 together create something genuinely new: a **private, offline, compounding knowledge system**. No OpenAI key. No cloud. No API keys. Gets smarter as you use it. Your own brain in a box.
</Aside>

## Phase 4 — vault-sync Engine <Badge text="In Progress" variant="caution" />

**Owner:** Fry · **Branch:** `spec/vault-sync-engine` · **Depends on:** Phase 3

Extends Quaid from a single-vault store to a multi-collection, file-system-aware knowledge engine. Memory stays current as you edit in Obsidian or any editor. Write safety guarantees keep SQLite state honest under concurrent agents.

**Landed so far:**
- Schema v5: `collections`, `file_state`, `embedding_jobs`
- Collections model: `quaid collection add/list/info/sync`
- `.quaidignore` with atomic validation
- Quarantine lifecycle for deleted pages
- Unix CAS write interlocks + crash-safe sentinels
- Live file watcher with 1.5s debounce (macOS/Linux)
- `memory_collections` MCP tool

**Still to ship:** IPC proxy (v0.14.0+), online restore handshake, Windows quarantine restore.

## Phase 5 — Conversation Memory + SLM Extraction <Badge text="Next" variant="note" />

<Aside type="note" title="Benchmark context">
  **LongMemEval:** Quaid 0.0% → Mem0 v3 93.4%  
  **LoCoMo:** Quaid 0.1% → Mem0 v3 91.6%  
  
  The gap is architectural, not a retrieval failure. Quaid stores whole conversation turns. LongMemEval asks specific factual questions. Without fact extraction at write time, retrieval cannot surface structured answers.
</Aside>

This phase closes the single biggest gap in agent memory quality. A local SLM runs fully airgapped — download once, never calls home.

<CardGrid>
  <Card title="[#137] Namespace isolation" icon="puzzle">
    <Badge text="No dependencies" variant="success" />

    Pure infrastructure. Per-namespace memory isolation for multi-agent and multi-session contexts. Also drops LongMemEval benchmark runtime from 3-6 hours to under 30 minutes.

    [→ GitHub #137](https://github.com/quaid-app/quaid/issues/137)
  </Card>
  <Card title="[#134] Large corpus performance" icon="rocket">
    <Badge text="No dependencies" variant="success" />

    Benchmark and optimize for 10K+ document stores. Current import: ~437s for 350 docs. Target: under 60s. Validate Quaid at the scale real users need (GBrain operates at 75K files).

    [→ GitHub #134](https://github.com/quaid-app/quaid/issues/134)
  </Card>
  <Card title="[#105] Conversation ingestion + SLM extraction" icon="star">
    <Badge text="Core feature" variant="tip" />

    Ingest conversation turns, extract structured facts using a local SLM, store as typed pages. Non-blocking background queue. Fully airgapped.

    **Recommended model:** Phi-3.5 Mini (MIT, ~2GB) as default. Configurable — swap to Gemma 3 1B/4B for quality.

    **Target:** LongMemEval > 40%, LoCoMo > 40% after implementation.

    [→ GitHub #105](https://github.com/quaid-app/quaid/issues/105)
  </Card>
  <Card title="[#135] Contradiction resolution" icon="warning">
    <Badge text="Depends on #105" variant="caution" />

    Supersede/version stale facts. "I used to work at X, now Y" resolves correctly rather than keeping both facts active. Version chains, temporal-latest query mode.

    [→ GitHub #135](https://github.com/quaid-app/quaid/issues/135)
  </Card>
</CardGrid>

## Phase 6 — Knowledge Graph <Badge text="Planned" variant="note" />

Once entities are extracted from conversations and documents, Quaid can build a real knowledge graph — and query it.

<CardGrid>
  <Card title="[#107] Entity extraction at write time" icon="document">
    <Badge text="Graph foundation" variant="tip" />

    Extract people, companies, and concepts at ingest time using zero-LLM heuristics. Coordinate with #105 — may share the same SLM extraction pipeline.

    [→ GitHub #107](https://github.com/quaid-app/quaid/issues/107)
  </Card>
  <Card title="[#72] Self-wiring knowledge graph" icon="random">
    <Badge text="Depends on #107" variant="caution" />

    YAML frontmatter as first-class graph edges. Documents auto-link via wikilinks and tags. From GBrain data: +28% graph search performance, -53% noisy results.

    [→ GitHub #72](https://github.com/quaid-app/quaid/issues/72)
  </Card>
  <Card title="[#133] Graph traversal query API" icon="magnifier">
    <Badge text="Depends on #72, #107" variant="caution" />

    Query by entity, not just text. `memory_graph_query(entity, hops)` — find all pages connected to X, what concepts link two people, 2-hop traversal in under 500ms.

    [→ GitHub #133](https://github.com/quaid-app/quaid/issues/133)
  </Card>
  <Card title="[#74] Multi-hop graph traversal" icon="right-arrow">
    <Badge text="Depends on #133" variant="caution" />

    Advanced graph queries: configurable 1-3 hop traversal, edge type weighting, relationship scoring combined with semantic similarity.

    [→ GitHub #74](https://github.com/quaid-app/quaid/issues/74)
  </Card>
</CardGrid>

## Phase 7 — Intelligence Layer <Badge text="Planned" variant="note" />

With the graph built and conversation memory working, these features make Quaid genuinely proactive.

<CardGrid>
  <Card title="[#136] Active memory enrichment" icon="information">
    <Badge text="Depends on #107, #133" variant="caution" />

    When new content arrives, existing entity memories update proactively. GBrain v0.23.0's headline feature: "Your brain actually gets enriched from your conversations."

    [→ GitHub #136](https://github.com/quaid-app/quaid/issues/136)
  </Card>
  <Card title="[#75] Noise reduction + deduplication" icon="list-format">
    Result deduplication and relevance filtering. Most valuable at scale. Can be implemented at any point but shines when the corpus is large.

    [→ GitHub #75](https://github.com/quaid-app/quaid/issues/75)
  </Card>
  <Card title="[#76] Context compression (REFRAG)" icon="compress">
    Compress retrieved chunks before sending to an LLM. Reduces tokens, reduces cost, maintains quality. High value at production scale.

    [→ GitHub #76](https://github.com/quaid-app/quaid/issues/76)
  </Card>
</CardGrid>

## Deferred

These are known design choices, not oversights.

| Feature | Status | Reasoning |
|---------|--------|-----------|
| npm, Homebrew, winget | Planned | Registry ownership + release automation not yet in place |
| WASM compilation | Possible | Strong Rust support, not yet a priority |
| Collaborative / multi-user | Not planned | Single-writer by design. No auth, RBAC, or CRDTs. |
| LLM-assisted contradiction detection | Deferred | Binary stays dumb. Cross-page reasoning in the `maintain` skill. |
| Room-level palace filtering | Deferred | Wing-only until benchmarks prove room-level helps. |
